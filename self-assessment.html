Introduction

Program Growth and Professional Direction

Throughout the Computer Science program, I moved from learning individual programming concepts to thinking about complete systems. Courses in programming, databases, algorithms, testing, full-stack development, and security helped me understand that professional software is more than code that runs. It also needs to be readable, maintainable, secure, testable, and useful to the people who depend on it.

Completing this ePortfolio helped me showcase that growth because it required me to revisit an existing project, evaluate its weaknesses, and improve it in several connected areas. The Travlr application gave me a practical way to demonstrate full-stack development skills: customer-facing pages, Express routes and controllers, API endpoints, MongoDB/Mongoose modeling, and an Angular administration interface. Enhancing the same application across multiple categories also helped me see how design choices in one layer affect the rest of the system.

My professional goal is to continue growing toward software engineering and full-stack web development roles. I want to build applications that solve real problems, communicate clearly with users and teammates, and handle data responsibly. This portfolio supports that goal by showing that I can take an existing codebase, improve it intentionally, and explain the value of those improvements to a technical or non-technical audience.

Core Strengths

Skills Demonstrated Across the Portfolio

Collaboration and Communication

Code review taught me to communicate about software in a structured way: describe the current behavior, identify risks, propose improvements, and connect those improvements to stakeholder needs. Even when working individually, I treated the project as something another developer or manager would need to understand later.

Software Engineering and Databases

The Travlr project strengthened my understanding of MVC structure, RESTful API design, Angular service integration, and MongoDB-backed persistence. I practiced separating responsibilities across routes, controllers, models, and views so the application is easier to maintain.

Algorithms and Data Structures

The algorithms enhancement helped me apply classroom concepts to real application behavior. Binary search, filtering, grouping, date-range logic, and Map-based caching showed how data structure choices can improve performance, flexibility, and user value.

Security Mindset

The project reinforced the need to validate input, handle missing or malformed data, protect database integrity, and avoid unsafe query behavior. Security is not a single feature added at the end; it is a habit that shapes each route, model, and data flow.

Portfolio Summary

How the Artifacts Fit Together

The three enhanced artifacts work together as one story of improving a full-stack application. The software design artifact focuses on the structure and user flow of the customer-facing Travlr experience. The algorithms and data structures artifact focuses on how trip data can be searched, filtered, grouped, cached, and retrieved more efficiently. The database artifact focuses on data integrity, API behavior, and administrative management of trip records.

Together, these artifacts demonstrate that I can evaluate an existing system, identify meaningful improvements, and implement changes that support both users and future developers. They also show that I can communicate technical work through written narratives, evidence links, and a GitHub Pages presentation that is organized for an outside audience.
