Artifact Description

What the Artifact Is

The selected artifact is the Travlr Getaways web application, created from coursework involving full-stack web development with Express, Handlebars, Angular, and MongoDB. The software design enhancement focuses on improving how customers move from a trip list to a specific trip record.

The original project showed trip content, but the listing experience did not have a dedicated trip detail route. The enhancement adds a clearer workflow: a user can open the travel page, select a trip, and view a focused details page with the trip name, code, dates, resort, price, image, and description.

Enhancement

What Changed

Customer Workflow

The travel list now links each trip to `/travel/:code`, and the trip detail view gives customers a dedicated page for one record.

MVC Routing

The `travel` route delegates work to a controller, and the controller retrieves data from the API instead of mixing data access into the view.

Error Handling

The detail controller handles missing codes, not-found trips, API failures, and server errors with clear status-aware responses.

Presentation

The Handlebars templates and CSS were updated so the enhanced flow remains readable and consistent with the Travlr design.
