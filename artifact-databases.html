Artifact 3: Databases

This artifact highlights the database layer that supports trip storage, retrieval, and structured query behavior for the Travlr application.

What changed

I strengthened the application’s data persistence layer by connecting the trip model to a fuller set of access patterns and API behavior. This makes the project more representative of a complete web application that depends on reliable database interaction.

  • Implemented a Mongoose-based trip model with validation and required fields.
  • Extended the API to support retrieval, filtering, grouping, and data range queries.
  • Used structured data handling to support application features in a more scalable way.

Evidence and supporting files

  • Trip model
  • Trip controller logic
  • Seed data

Reflection

The database enhancement reinforced the importance of thoughtful schema design and controlled data access. I learned that data services are not just about storing information; they must support the application’s purpose safely and effectively.
